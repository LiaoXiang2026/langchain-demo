```python
class DocumentPipeline:
    def __init__(self, persist_dir=None, embedding_model=None, chunk_size=None, chunk_overlap=None):
        # ...
        self._store = VectorStore(persist_dir=self.persist_dir, embedding_model=self.embedding_model)

    def ingest(self, file_path: str) -> dict:
        """处理单个文档:加载→分块→存入向量库。"""
        filename = Path(file_path).name
        docs = load_document(file_path)
        chunks = split_documents(docs, self.chunk_size, self.chunk_overlap)
        count = self._store.add_documents(chunks, filename=filename)
        return {"filename": filename, "chunk_count": count}
```

From src/rag/vectorstore.py:
```python
def add_documents(self, docs: list[Document], filename: str) -> int:
    """添加文档到向量存储,返回 chunk 数量。"""
    for i, doc in enumerate(docs):
        doc.metadata["source"] = filename
        doc.metadata["chunk_id"] = i
    self._store.add_documents(docs)
    return len(docs)
```

修改 `src/rag/vectorstore.py` 的 `add_documents(self, docs, filename)` 方法:

当前实现(有问题):
```python
for i, doc in enumerate(docs):
    doc.metadata["source"] = filename
    doc.metadata["chunk_id"] = i
self._store.add_documents(docs)
```

改为——保留 source + chunk_id 注入逻辑,**额外**继承 doc 自带的 metadata 字段(因为 Plan 02 / Plan 03 会在上游传入 content_hash/title/author/publish_date/source_path)。但 source 和 chunk_id 必须仍然由本函数覆盖,避免上游传错。

正确实现:
```python
for i, doc in enumerate(docs):
    # 隔离 doc 自带 metadata,避免上游传入的 dict 引用在多次 add_documents 调用间共享
    # (LangChain Document 默认 metadata 是 dict 引用,不显式 copy 会让 chunk 间的元数据互相污染)
    doc.metadata = dict(doc.metadata)
    doc.metadata["source"] = filename
    doc.metadata["chunk_id"] = i
self._store.add_documents(docs)
```

为什么需要 `doc.metadata = dict(doc.metadata)` 这一步:每个 chunk 显式分配新 dict,避免上游传入的 dict 引用在多次 add_documents 调用间共享导致 cross-contamination(Phase 1 没踩的坑,Phase 2 因为要继承 content_hash/title/author 等字段,必须处理)。

**关键约束**: 不要改 `add_documents` 的签名,不要改 ChromaDB 写入的 API 调用,不要动 `search`/`delete_by_filename`/`list_documents`。
