<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 浏览器 → 后端 (POST /knowledge/upload-batch) | 用户通过 webkitdirectory 选中的文件夹,所有文件名是 untrusted |
| multipart filename → 临时目录 | file.filename 来自 multipart Content-Disposition,可能含 路径穿越 |
| 临时目录 → pipeline.ingest_batch | 临时文件走 Plan 02 ingest_batch,已有错误隔离和去重 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Elevation | multipart filename 含 ../../etc/passwd 路径穿越 | mitigate | safe_name = Path(filename).name 只取 basename;后端不依赖 webkitRelativePath 拼接到 upload_dir 之外 |
| T-02-12 | DoS | 一次上传 1000+ 文件撑爆内存 | accept | 200 篇规模场景(D-19);本 plan 不实现 max-files 限制 |
| T-02-13 | DoS | 巨大单文件(>50MB HTML) | accept | Pitfall 6 建议 50MB 限制但本 plan 不实现 |
| T-02-14 | Tampering | 上传 .html 含恶意 script | mitigate | Plan 01 cleaner 已剥 script |
| T-02-15 | Information Disclosure | 临时目录残留含敏感 HTML | mitigate | try/finally + shutil.rmtree(tmpdir, ignore_errors=True) 强制清理 |
| T-02-16 | DoS | 损坏文件导致整批失败 | mitigate | try/except 包每文件,errors 数组收集,不中断批次 |

| V12 File Handling | yes | 临时目录 + basename 防护 + cleanup |
| V13 API Security | yes | List[UploadFile] = File(...) FastAPI 校验 |
