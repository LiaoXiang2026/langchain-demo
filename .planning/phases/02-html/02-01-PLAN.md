```python
LOADER_MAP: dict[str, Callable[[str], BaseLoader]] = {
    ".txt": lambda path: TextLoader(path, encoding="utf-8"),
    ".md": lambda path: TextLoader(path, encoding="utf-8"),
    ".pdf": lambda path: PyMuPDFLoader(path),
    ".docx": lambda path: Docx2txtLoader(path),
    ".xlsx": lambda path: UnstructuredExcelLoader(path),
    ".xls": lambda path: UnstructuredExcelLoader(path),
}

def load_document(file_path: str) -> list[Document]:
    """加载单个文档,返回 Document 列表。"""
    ext = Path(file_path).suffix.lower()
    loader_factory = LOADER_MAP.get(ext)
    if not loader_factory:
        raise ValueError(f"不支持的文件格式: {ext}。支持的格式: {SUPPORTED_FORMATS}")
    loader = loader_factory(file_path)
    return loader.load()
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 用户 → 服务端 (file upload) | 用户上传的 .html 文件是 untrusted input,可能含恶意 `<script>`、base64 图片、巨文件 |
| Fixture 文件 → cleaner | 清洗器处理 fixtures 时,fixtures 自身是 trusted (我们写的) |
| 清洗器 → VectorStore | 清洗后 text 进向量库,但本 plan 不写向量库(留给 Plan 02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `<script>` 注入到清洗后文本 | mitigate | cleaner 必须 `decompose()` 所有 `<script>`/`<noscript>`/`<style>`/`<iframe>` 标签(D-10),不只是 strip 文本 |
| T-02-02 | DoS | 巨大 base64 图片撑爆 chunk | mitigate | `<img>` 处理时 `img.attrs = {"alt": ..., "data-img": "[图片]"}` 必须剥 src(见 Task 2 行为 6) |
| T-02-03 | DoS | 巨大 HTML 文件 ( > 50MB) | accept | 本 plan 只处理单文件加载,大文件限制由 API 层 (Plan 03) 控制 |
| T-02-04 | Information Disclosure | GBK 误识别为 UTF-8 导致内存错乱 | mitigate | `chardet.detect` 后归一化到 `gb18030`,`errors="replace"` 兜底,绝不抛 UnicodeDecodeError |
| T-02-05 | DoS | lxml 解析 OOM (恶意 billion-laughs) | accept | lxml 默认禁用外部实体;200 篇场景的 fixture 都是真实文章大小,无此威胁 |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck 已对 beautifulsoup4/lxml/chardet 三个包做合法性审计(02-RESEARCH.md Package Legitimacy Audit),均 [OK] 来自 PyPI 主流维护者 |
